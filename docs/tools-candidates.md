## get_location_history — Google Map タイムライン

その日の移動・訪問場所を取得する。

### データソース

Google Maps タイムライン（ロケーション履歴）

### 取得方法の選択肢

| 方法 | 概要 | 実現性 |
|---|---|---|
| スマホからエクスポート | Google Maps アプリ → Your timeline → Export Timeline data で `Timeline.json` をデバイスに保存 | 手動操作。都度エクスポートが必要 |
| Google Takeout | 以前は Takeout で全履歴をダウンロードできた | 2025 年のプライバシー更新以降、タイムラインデータはデバイスローカルに移行。Takeout からは取得不可 |
| KML エクスポート（Cookie 利用） | Web 版タイムラインから日単位で KML をダウンロード。Cookie を使った自動化ツールあり | 非公式。Cookie の有効期限管理が必要。Google のサイト変更で壊れやすい |

### 制約・課題

- **公式 API は存在しない**（Google Maps Platform に Location History API はない）
- 2025 年以降、タイムラインデータはデバイスローカル保存に変更された。クラウドからのプログラマティックアクセスは事実上不可能に
- エクスポートされるデータは JSON 形式で、`semanticSegments`（訪問場所・移動経路）を含む
- KML 方式は Cookie ベースのスクレイピングで、安定性・保守性に難あり

### 実装アプローチ案

スマホからエクスポートした `Timeline.json` を定期的にサーバーに配置し、それを MCP ツールが読み取る方式。完全自動化は難しく、手動エクスポートの運用フローが必要。

### 難易度

高（公式 API なし。デバイスローカル化により自動取得が困難）

## get_transactions — Money Forward 決済履歴

その日の決済・支出履歴を取得する。

### データソース

マネーフォワード ME（個人向け家計簿アプリ）

### 取得方法の選択肢

| 方法 | 概要 | 実現性 |
|---|---|---|
| マネーフォワード ME 公式 API | `moneyforward/api-doc`（GitHub）にドキュメントあり。OAuth2.0 認証 | 個人開発者向けに公開されているが、利用申請・審査が必要な可能性あり。要確認 |
| CSV エクスポート | アプリ / Web から入出金明細を CSV でダウンロード可能 | 手動。自動化不可 |
| スクレイピング | Selenium 等で Web 版にログインしてデータ取得。Qiita 等に実例あり | 非公式。2FA やサイト変更で壊れやすい。セキュリティリスクあり |
| 非公式 REST API | `sammrai/mfapi`（GitHub）にリバースエンジニアリングされた API ドキュメントあり | 非公式のため安定性の保証なし。認証トークンの取得方法も非公式 |

### 制約・課題

- マネーフォワード ME（個人向け）の公式 API は、外部開発者に広く公開されているわけではない（クラウド会計等の法人向け API とは別）
- 非公式 API は存在するが、利用規約違反のリスクあり
- CSV エクスポートなら確実だが、手動になる
- 取得できるデータ：日付、金額、カテゴリ、内容（メモ）、口座名など

### 実装アプローチ案

まずは公式 API の利用可否を確認。利用できない場合は、定期的に CSV エクスポート → サーバーに配置 → MCP ツールが読み取る方式。非公式 API は最終手段。

### 難易度

中（公式 API が使えれば中、使えなければ高）

## get_watch_history — YouTube 視聴履歴

その日に見た動画の一覧を取得する。

### データソース

YouTube（Google アカウント）

### 取得方法の選択肢

| 方法 | 概要 | 実現性 |
|---|---|---|
| Google Takeout | YouTube の視聴履歴を JSON / HTML でエクスポート可能。動画 ID・タイトル・チャンネル名・タイムスタンプを含む | 手動操作。ただし定期エクスポートは可能 |
| YouTube Data API v3 | 動画のメタデータ（タイトル・サムネイル・再生時間等）を取得可能。OAuth 2.0 認証 | **視聴履歴の直接取得 API は 2017 年に廃止済み**。Takeout のデータに対して補完的に使う |
| My Activity ページ | Google の My Activity から YouTube 視聴履歴を確認可能 | API なし。スクレイピングは非推奨 |

### 制約・課題

- YouTube Data API v3 で視聴履歴を直接取得する方法は 2017 年に廃止された
- Google Takeout 経由が実質唯一の方法。ただし手動エクスポートが必要
- Takeout のタイムスタンプは不正確な場合がある（タイムゾーン処理の問題）
- 削除済み動画は ID のみでタイトル等が欠落する（約 15%）
- Takeout → YouTube Data API v3 で動画メタデータを補完する二段構成が一般的
- API のデフォルトクォータは 10,000 ポイント/日（動画情報取得は 1 件あたり約 11 ポイント → 約 900 件/日）

### 実装アプローチ案

1. Google Takeout で定期的に視聴履歴をエクスポート（JSON）
2. MCP ツールがその JSON をパースし、日付でフィルタ
3. 必要に応じて YouTube Data API v3 で動画メタデータ（サムネイル・再生時間等）を補完

### 難易度

中（Takeout 依存だが、データ構造は明確。API 補完も容易）

## 実装優先度まとめ

| ツール | 難易度 | 自動化 | 振り返り価値 | 優先度 |
|---|---|---|---|---|
| `get_watch_history` | 中 | △（Takeout 手動） | 中 | ★★★ |
| `get_transactions` | 中〜高 | △（API 要調査） | 高 | ★★★ |
| `get_location_history` | 高 | ✕（手動エクスポート） | 高 | ★★ |

> 振り返り価値が高くても自動化が難しいものは優先度を下げている。
> 公式 API の有無が実装コストに直結するため、まず Money Forward ME の API 利用可否を調査するのが次のアクション。
